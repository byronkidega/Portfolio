Flask Case Management System

Overview

This project is a web-based case management system developed with Flask, SQLAlchemy, and SQLite. It provides role-based access control and robust user authentication. The auth.py module handles user registration, login, role-based redirects, analysis of uploaded reports, and detailed logging of activities conducted within the system. It supports distinct roles like Viewer, Investigative Officer, Admin, and Super User.

Main Features

  • User Authentication: Login with role-based access control.
  • File Uploads: Upload reports and case files with metadata.
  • Search Reports: Search reports by reference number or case name.
  • Report Reference Number: Automatically generated for each report.
  • Analytical Dashboard: Augmented Intelligence for trend analysis of uploaded reports.
  • Admin Features: Manage users, view deactivated user logs, and more.
  • Soft Deletion: Soft delete users and retain their data for auditing.
  • Audit Logs: Track user deletions and system events.
  • Excel Export Functions: Export logs and reports to Excel files.

User Registration and Role-Based Activities

Authentication and Authorization:
Using Flask-Login, users are authenticated and assigned roles, allowing for role-based redirection and content access restriction. Only authorized users can view certain pages or perform specific actions.

  • Validation: Validate user signup input such as name length, email uniqueness, and password strength.
  • Password Encryption: User-entered passwords are encrypted using Flaskâ€™s generate_password_hash.
  • Role Validation: Ensure users can only select authorized roles.
  • Role-Based Redirects: Users are redirected to appropriate pages according to their roles.

Monitoring and Logging

The application logs critical activities, including:

  • Deleted Reports: Tracks reports deleted by investigative officers.
  • User List: Lists active users with relevant metadata.
  • Deactivated Users: Lists deactivated users with details of the deactivator.
  • Reactivated Users: Lists reactivated users with details of the reactivator.

File and Database Management

  • SQLite Database: SQLAlchemy ORM is used for database interactions, offering protection against SQL injection.
  • File Storage: Uploaded files are stored securely with unique filenames derived from reference numbers, ensuring integrity.

Security Features

  • Password Hashing: User passwords are hashed before storage using SHA256 with generate_password_hash.
  • Role-Based Access Control: Access is restricted based on user roles.
  • Input Validation: Protect against SQL injection and input-based attacks.
  • CSRF Protection: Built-in Flask CSRF protection is enabled on forms.
